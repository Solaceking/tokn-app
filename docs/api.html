TOKN API Reference

REST API endpoints for integrating TOKN with your applications.

Authentication: All API requests require a valid API key. Generate keys in Settings → API Keys.

Authentication

Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY

Or as a query parameter:

GET /api/tokens?apiKey=YOUR_API_KEY

Rate Limiting

  • Free tier: 100 requests/hour
  • Pro tier: 1,000 requests/hour
  • Exceeding limits returns HTTP 429 (Too Many Requests)

Endpoints

Tokens

List Tokens

GET /api/tokens

Response:

{
  "tokens": [
    {
      "id": "token_123",
      "service": "OpenAI",
      "description": "Production API key",
      "category": "AI/ML",
      "status": "ACTIVE",
      "createdAt": "2024-01-01T00:00:00Z",
      "updatedAt": "2024-01-01T00:00:00Z"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 50
}

Get Token

GET /api/tokens/{id}

Response:

{
  "id": "token_123",
  "service": "OpenAI",
  "token": "encrypted_token_data",
  "description": "Production API key",
  "category": "AI/ML",
  "status": "ACTIVE",
  "createdAt": "2024-01-01T00:00:00Z",
  "updatedAt": "2024-01-01T00:00:00Z"
}

Create Token

POST /api/tokens

Request Body:

{
  "service": "OpenAI",
  "token": "sk-abc123",
  "description": "Production API key",
  "category": "AI/ML"
}

Update Token

PUT /api/tokens/{id}

Request Body:

{
  "description": "Updated description",
  "category": "Development"
}

Delete Token

DELETE /api/tokens/{id}

Token Operations

Test Token

POST /api/tokens/{id}/test

Response:

{
  "success": true,
  "message": "Token is valid",
  "statusCode": 200,
  "responseTime": 150
}

Decrypt Token

POST /api/tokens/{id}/decrypt

Response:

{
  "token": "sk-abc123",
  "validFor": 300  // Seconds until token re-encryption
}
Security Note: Decrypted tokens are only available for 5 minutes before being re-encrypted.

Parser API

Parse Text for Tokens

POST /api/parse

Request Body:

{
  "text": "OPENAI_API_KEY=sk-abc123\nGITHUB_TOKEN=ghp_xyz789",
  "autoSave": false
}

Response:

{
  "tokens": [
    {
      "service": "OpenAI",
      "token": "sk-abc123",
      "confidence": 0.95
    },
    {
      "service": "GitHub",
      "token": "ghp_xyz789",
      "confidence": 0.92
    }
  ]
}

AI-Powered Parser

POST /api/parse/ai

Request Body:

{
  "text": "My OpenAI key is sk-abc123 and GitHub token is ghp_xyz789",
  "model": "gpt-4"
}

Activities

List Activities

GET /api/activities

Query Parameters:

  • action: Filter by action type
  • service: Filter by service
  • from: Start date (ISO format)
  • to: End date (ISO format)
  • limit: Results per page (default: 50)
  • page: Page number (default: 1)

Error Handling

Error Codes

Code Description
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing API key
403 Forbidden - Insufficient permissions
404 Not Found - Resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error

Error Response Format

{
  "error": "Invalid API key",
  "code": "INVALID_API_KEY",
  "details": "The provided API key is invalid or expired"
}

Client Libraries

JavaScript/Node.js

npm install tokn-client
import { ToknClient } from 'tokn-client';

const client = new ToknClient({
  apiKey: 'YOUR_API_KEY',
  baseUrl: 'https://your-tokn-instance.com'
});

// List tokens
const tokens = await client.tokens.list();

// Create token
const newToken = await client.tokens.create({
  service: 'OpenAI',
  token: 'sk-abc123',
  description: 'Production key'
});

Python

pip install tokn-python
from tokn import ToknClient

client = ToknClient(api_key="YOUR_API_KEY")

# List tokens
tokens = client.tokens.list()

# Test a token
result = client.tokens.test("token_id")

cURL Examples

# List tokens
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://your-tokn-instance.com/api/tokens

# Create token
curl -X POST -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"service":"OpenAI","token":"sk-abc123"}' \
  https://your-tokn-instance.com/api/tokens
